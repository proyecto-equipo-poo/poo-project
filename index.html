1. Clases Abstractas e Interfaces

Las clases abstractas e interfaces son pilares fundamentales de la Programación Orientada a Objetos. Permiten aplicar abstracción, reutilización de código y polimorfismo.

Concepto

Una clase abstracta es una clase que no puede ser instanciada directamente y sirve como base para otras clases. Las interfaces definen contratos que las clases deben implementar.

Características

  • Permiten reutilización de código.
  • Aplican abstracción.
  • Facilitan el polimorfismo.
  • Organizan mejor el sistema.

Ejemplo 1 - Clase Abstracta

abstract class Animal {

    protected String nombre;

    public Animal(String nombre) {
        this.nombre = nombre;
    }

    public abstract void hacerSonido();
}

class Perro extends Animal {

    public Perro(String nombre) {
        super(nombre);
    }

    @Override
    public void hacerSonido() {
        System.out.println("Guau");
    }
}

La clase Animal define un comportamiento general. La clase Perro hereda e implementa el método abstracto.

Ejemplo 2 - Interfaces

interface Pagable {

    void pagar();
}

class Factura implements Pagable {

    @Override
    public void pagar() {
        System.out.println("Pago realizado");
    }
}

La interfaz define un contrato que debe ser implementado por cualquier clase relacionada con pagos.

Ventajas

  • Mayor reutilización.
  • Mejor mantenimiento.
  • Arquitectura más organizada.
  • Flexibilidad en el diseño.

Ejercicios Propuestos

  • Crear una clase abstracta llamada Vehiculo con un método abstracto acelerar().
  • Crear una interfaz llamada Conectable para dispositivos electrónicos.

Ejemplo 1

}

Ejemplo 2

3. Manejo de Errores y Excepciones

Las excepciones permiten controlar errores durante la ejecución del programa.

Ejemplo 1

try{
    int n = 10 / 0;
}catch(Exception e){
    System.out.println(e);
}

Ejemplo 2

throw new Exception("Error");
