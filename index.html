Programación Orientada a Objetos

Sitio académico sobre los temas fundamentales de POO en Java.

1. Clases y Objetos

Las clases son plantillas que permiten crear objetos. Los objetos representan entidades con atributos y métodos.

Ejemplo 1


class Persona {

    String nombre;

    void saludar() {
        System.out.println("Hola");
    }
}

Ejemplo 2


class Vehiculo {

    String marca;
    int velocidad;
}

Ejercicios

  • Crear una clase Estudiante con atributos y métodos.
  • Diseñar una clase Producto para una tienda virtual.

2. Composición y Modularización del Código

La composición es un principio de la Programación Orientada a Objetos donde una clase contiene objetos de otras clases para construir funcionalidades más complejas.

La modularización consiste en dividir un sistema en pequeñas partes independientes llamadas módulos, facilitando la organización, reutilización y mantenimiento del código.

Definición

En POO, la composición representa una relación del tipo "tiene un". Por ejemplo, un automóvil tiene un motor. Esto permite crear sistemas más organizados y reutilizables.

Características

  • Favorece la reutilización del código.
  • Permite construir sistemas complejos.
  • Mejora la organización del software.
  • Reduce la dependencia entre clases.
  • Facilita el mantenimiento del código.

Ejemplo Resuelto


class Motor {

    void encender() {

        System.out.println("Motor encendido");
    }
}

class Auto {

    private Motor motor;

    public Auto() {

        motor = new Motor();
    }

    void arrancar() {

        motor.encender();

        System.out.println("Auto en movimiento");
    }
}

public class Main {

    public static void main(String[] args) {

        Auto auto = new Auto();

        auto.arrancar();
    }
}

Explicación del Ejemplo

La clase Auto utiliza un objeto de la clase Motor. Esto demuestra composición porque el automóvil "tiene un" motor.

Gracias a este principio de POO, el sistema queda más organizado y cada clase tiene una responsabilidad específica.

Ventajas

  • Mayor reutilización de clases.
  • Mejor mantenimiento del sistema.
  • Mayor escalabilidad del software.
  • Separación clara de responsabilidades.

Ejercicios Propuestos

  • Crear una clase Computador que tenga una clase Procesador utilizando la composición.
  • Diseñar un sistema de biblioteca usando clases Libro, Usuario y Prestamo.
