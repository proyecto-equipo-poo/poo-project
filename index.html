1. Clases Abstractas e Interfaces

Las clases abstractas y las interfaces son conceptos fundamentales de la Programación Orientada a Objetos. Permiten aplicar abstracción y polimorfismo, facilitando la reutilización del código y la organización de sistemas complejos.

¿Qué es una clase abstracta?

Una clase abstracta es una clase que no puede ser instanciada directamente y sirve como base para otras clases. Puede contener métodos abstractos y métodos normales.

Ejemplo 1 - Clase Abstracta


abstract class Animal {

    abstract void sonido();
}

class Perro extends Animal {

    void sonido() {
        System.out.println("Guau");
    }
}

public class Main {

    public static void main(String[] args) {

        Perro perro = new Perro();
        perro.sonido();
    }
}

¿Qué es una interfaz?

Una interfaz define comportamientos que otras clases deben implementar. En Java se utiliza la palabra reservada interface.

Ejemplo 2 - Interfaz


interface Vehiculo {

    void acelerar();
}

class Carro implements Vehiculo {

    public void acelerar() {
        System.out.println("El carro acelera");
    }
}

public class Main {

    public static void main(String[] args) {

        Carro carro = new Carro();
        carro.acelerar();
    }
}

2. Composición y Modularización del Código

La composición es una técnica de Programación Orientada a Objetos donde una clase contiene objetos de otras clases para construir funcionalidades más complejas.

La modularización permite dividir el código en partes pequeñas, organizadas y reutilizables, facilitando el mantenimiento y escalabilidad del software.

¿Qué es la composición?

En composición, una clase utiliza otra clase como atributo. Esto representa relaciones del tipo "tiene un".

Ejemplo 1 - Composición


class Motor {

    void encender() {
        System.out.println("Motor encendido");
    }
}

class Carro {

    private Motor motor = new Motor();

    void arrancar() {
        motor.encender();
    }
}

public class Main {

    public static void main(String[] args) {

        Carro carro = new Carro();
        carro.arrancar();
    }
}

¿Qué es la modularización?

La modularización organiza el código en diferentes clases y métodos independientes para mejorar la claridad y reutilización del sistema.

Ejemplo 2 - Modularización


class Calculadora {

    int sumar(int a, int b) {
        return a + b;
    }

    int restar(int a, int b) {
        return a - b;
    }
}

public class Main {

    public static void main(String[] args) {

        Calculadora calc = new Calculadora();

        System.out.println(calc.sumar(5, 3));
        System.out.println(calc.restar(10, 4));
    }
}
