Programación Orientada a Objetos

Sitio académico sobre los temas fundamentales de POO en Java.

1. Clases y Objetos

Las clases son plantillas que permiten crear objetos. Los objetos representan entidades con atributos y métodos.

Ejemplo 1


class Persona {

    String nombre;

    void saludar() {
        System.out.println("Hola");
    }
}

Ejemplo 2


class Vehiculo {

    String marca;
    int velocidad;
}

Ejercicios

  • Crear una clase Estudiante con atributos y métodos.
  • Diseñar una clase Producto para una tienda virtual.

2. Composición y Modularización del Código

La composición es un principio de la Programación Orientada a Objetos donde una clase contiene objetos de otras clases para construir funcionalidades más complejas.

La modularización consiste en dividir un sistema en pequeñas partes independientes llamadas módulos, facilitando la organización, reutilización y mantenimiento del código.

Definición

En POO, la composición representa una relación del tipo "tiene un". Por ejemplo, un automóvil tiene un motor. Esto permite crear sistemas más organizados y reutilizables.

Características

  • Favorece la reutilización del código.
  • Permite construir sistemas complejos.
  • Mejora la organización del software.
  • Reduce la dependencia entre clases.
  • Facilita el mantenimiento del código.

Ejemplo Resuelto


class Motor {

    void encender() {

        System.out.println("Motor encendido");
    }
}

class Auto {

    private Motor motor;

    public Auto() {

        motor = new Motor();
    }

    void arrancar() {

        motor.encender();

        System.out.println("Auto en movimiento");
    }
}

public class Main {

    public static void main(String[] args) {

        Auto auto = new Auto();

        auto.arrancar();
    }
}

Explicación del Ejemplo

La clase Auto utiliza un objeto de la clase Motor. Esto demuestra composición porque el automóvil "tiene un" motor.

Gracias a este principio de POO, el sistema queda más organizado y cada clase tiene una responsabilidad específica.

Ventajas

  • Mayor reutilización de clases.
  • Mejor mantenimiento del sistema.
  • Mayor escalabilidad del software.
  • Separación clara de responsabilidades.

Ejercicios Propuestos

  • Crear una clase Computador que tenga una clase Procesador utilizando la composición.
  • Diseñar un sistema de biblioteca usando clases Libro, Usuario y Prestamo.

3. Manejo de Errores y Excepciones

El manejo de errores y excepciones es fundamental en la Programación Orientada a Objetos porque permite controlar situaciones inesperadas durante la ejecución del programa.

En Java, las excepciones ayudan a evitar que el sistema finalice de forma incorrecta, mejorando la estabilidad y seguridad del software.

Definición

Una excepción es un evento que ocurre durante la ejecución de un programa y altera el flujo normal de las instrucciones.

Java utiliza las palabras reservadas try, catch y finally para manejar excepciones.

Características

  • Permiten controlar errores de ejecución.
  • Evitan que el programa termine inesperadamente.
  • Facilitan la depuración del sistema.
  • Mejoran la seguridad y estabilidad del software.
  • Permiten separar la lógica del manejo de errores.

Ejemplo Resuelto


public class Main {

    public static void main(String[] args) {

        try {

            int resultado = 10 / 0;

            System.out.println(resultado);

        } catch (ArithmeticException e) {

            System.out.println("Error: division por cero");

        } finally {

            System.out.println("Fin del programa");
        }
    }
}

Explicación del Ejemplo

En el ejemplo se intenta realizar una división entre cero, lo que genera una excepción de tipo ArithmeticException.

El bloque catch captura el error y evita que el programa termine de manera abrupta.

El bloque finally se ejecuta siempre, independientemente de si ocurre o no una excepción.

Ventajas

  • Mayor estabilidad del software.
  • Mejor experiencia para el usuario.
  • Facilita el mantenimiento del sistema.
  • Permite detectar errores de manera controlada.
  • Mejora la calidad del código.

Relación con POO

El manejo de excepciones se relaciona con POO porque las excepciones en Java son objetos que pertenecen a clases específicas.

Además, permite crear sistemas más robustos, organizados y mantenibles.

Ejercicios Propuestos

  • Crear un programa que capture un error al ingresar texto en una variable numérica.
  • Diseñar un sistema que valide si un usuario ingresó correctamente una contraseña.

4. Colecciones y Estructuras de Datos

Las colecciones y estructuras de datos permiten almacenar, organizar y manipular grandes cantidades de información dentro de un programa.

En Java, las colecciones forman parte del Java Collections Framework, el cual proporciona clases y métodos reutilizables para trabajar con datos de manera eficiente.

Definición

Una colección es un objeto que permite almacenar múltiples elementos en una sola estructura.

Algunas colecciones comunes en Java son: ArrayList, LinkedList, HashSet y HashMap.

Características

  • Permiten almacenar múltiples objetos.
  • Facilitan la búsqueda y organización de datos.
  • Son dinámicas y flexibles.
  • Reducen la complejidad del código.
  • Forman parte de la biblioteca estándar de Java.

Ejemplo Resuelto


import java.util.ArrayList;

public class Main {

    public static void main(String[] args) {

        ArrayList estudiantes = new ArrayList<>();

        estudiantes.add("Juan");
        estudiantes.add("Maria");
        estudiantes.add("Carlos");

        for (String nombre : estudiantes) {

            System.out.println(nombre);
        }
    }
}

Explicación del Ejemplo

En este ejemplo se utiliza un ArrayList para almacenar nombres de estudiantes.

El método add() permite agregar elementos a la colección, mientras que el ciclo for-each recorre cada objeto almacenado.

Esto demuestra cómo las colecciones facilitan el manejo dinámico de datos en Java.

Ventajas

  • Mayor organización de datos.
  • Facilidad para agregar y eliminar elementos.
  • Mayor reutilización de código.
  • Mejor rendimiento en aplicaciones grandes.
  • Facilitan el desarrollo de sistemas complejos.

Relación con POO

Las colecciones trabajan directamente con objetos, lo cual es fundamental en Programación Orientada a Objetos.

Por ejemplo, un ArrayList puede almacenar objetos creados a partir de una clase específica.

Esto permite organizar entidades del sistema de manera eficiente y reutilizable.

Ejercicios Propuestos

  • Crear un ArrayList que almacene objetos de una clase Producto.
  • Diseñar un sistema de biblioteca utilizando colecciones para almacenar libros y usuarios.
