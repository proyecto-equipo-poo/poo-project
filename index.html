2. Composición y Modularización del Código

La composición es un principio de la Programación Orientada a Objetos donde una clase contiene objetos de otras clases para construir funcionalidades más complejas.

La modularización consiste en dividir un sistema en pequeñas partes independientes llamadas módulos, facilitando la organización, reutilización y mantenimiento del código.

Definición

En POO, la composición representa una relación del tipo "tiene un". Por ejemplo, un automóvil tiene un motor. Esto permite crear sistemas más organizados y reutilizables.

Características

  • Favorece la reutilización del código.
  • Permite construir sistemas complejos.
  • Mejora la organización del software.
  • Reduce la dependencia entre clases.
  • Facilita el mantenimiento del código.

Ejemplo Resuelto


class Motor {

    void encender() {

        System.out.println("Motor encendido");
    }
}

class Auto {

    private Motor motor;

    public Auto() {

        motor = new Motor();
    }

    void arrancar() {

        motor.encender();

        System.out.println("Auto en movimiento");
    }
}

public class Main {

    public static void main(String[] args) {

        Auto auto = new Auto();

        auto.arrancar();
    }
}

Explicación del Ejemplo

La clase Auto utiliza un objeto de la clase Motor. Esto demuestra composición porque el automóvil "tiene un" motor.

Gracias a este principio de POO, el sistema queda más organizado y cada clase tiene una responsabilidad específica.

Ventajas

  • Mayor reutilización de clases.
  • Mejor mantenimiento del sistema.
  • Mayor escalabilidad del software.
  • Separación clara de responsabilidades.

Ejercicios Propuestos

  • Crear una clase Computador que tenga una clase Procesador utilizando la composición.
  • Diseñar un sistema de biblioteca usando clases Libro, Usuario y Prestamo.

3. Manejo de Errores y Excepciones

El manejo de errores y excepciones es fundamental en la Programación Orientada a Objetos porque permite controlar situaciones inesperadas durante la ejecución del programa.

En Java, las excepciones ayudan a evitar que el sistema finalice de forma incorrecta, mejorando la estabilidad y seguridad del software.

Definición

Una excepción es un evento que ocurre durante la ejecución de un programa y altera el flujo normal de las instrucciones.

Java utiliza las palabras reservadas try, catch y finally para manejar excepciones.

Características

  • Permiten controlar errores de ejecución.
  • Evitan que el programa termine inesperadamente.
  • Facilitan la depuración del sistema.
  • Mejoran la seguridad y estabilidad del software.
  • Permiten separar la lógica del manejo de errores.

Ejemplo Resuelto


public class Main {

    public static void main(String[] args) {

        try {

            int resultado = 10 / 0;

            System.out.println(resultado);

        } catch (ArithmeticException e) {

            System.out.println("Error: division por cero");

        } finally {

            System.out.println("Fin del programa");
        }
    }
}

Explicación del Ejemplo

En el ejemplo se intenta realizar una división entre cero, lo que genera una excepción de tipo ArithmeticException.

El bloque catch captura el error y evita que el programa termine de manera abrupta.

El bloque finally se ejecuta siempre, independientemente de si ocurre o no una excepción.

Ventajas

  • Mayor estabilidad del software.
  • Mejor experiencia para el usuario.
  • Facilita el mantenimiento del sistema.
  • Permite detectar errores de manera controlada.
  • Mejora la calidad del código.

Relación con POO

El manejo de excepciones se relaciona con POO porque las excepciones en Java son objetos que pertenecen a clases específicas.

Además, permite crear sistemas más robustos, organizados y mantenibles.

Ejercicios Propuestos

  • Crear un programa que capture un error al ingresar texto en una variable numérica.
  • Diseñar un sistema que valide si un usuario ingresó correctamente una contraseña.

4. Colecciones y Estructuras de Datos

Las colecciones y estructuras de datos permiten almacenar, organizar y manipular grandes cantidades de información dentro de un programa.

En Java, las colecciones forman parte del Java Collections Framework, el cual proporciona clases y métodos reutilizables para trabajar con datos de manera eficiente.

Definición

Una colección es un objeto que permite almacenar múltiples elementos en una sola estructura.

Algunas colecciones comunes en Java son: ArrayList, LinkedList, HashSet y HashMap.

Características

  • Permiten almacenar múltiples objetos.
  • Facilitan la búsqueda y organización de datos.
  • Son dinámicas y flexibles.
  • Reducen la complejidad del código.
  • Forman parte de la biblioteca estándar de Java.

Ejemplo Resuelto


ArrayList<String> estudiantes = new ArrayList<>();

estudiantes.add("Ana");
estudiantes.add("Carlos");

for (String nombre : estudiantes) {

    System.out.println(nombre);

}

Explicación del Ejemplo

En este ejemplo se utiliza un ArrayList para almacenar nombres de estudiantes.

El método add() permite agregar elementos a la colección, mientras que el ciclo for-each recorre cada objeto almacenado.

Esto demuestra cómo las colecciones facilitan el manejo dinámico de datos en Java.

Ventajas

  • Mayor organización de datos.
  • Facilidad para agregar y eliminar elementos.
  • Mayor reutilización de código.
  • Mejor rendimiento en aplicaciones grandes.
  • Facilitan el desarrollo de sistemas complejos.

Relación con POO

Las colecciones trabajan directamente con objetos, lo cual es fundamental en Programación Orientada a Objetos.

Por ejemplo, un ArrayList puede almacenar objetos creados a partir de una clase específica.

Esto permite organizar entidades del sistema de manera eficiente y reutilizable.

Ejercicios Propuestos

  • Crear un ArrayList que almacene objetos de una clase Producto.
  • Diseñar un sistema de biblioteca utilizando colecciones para almacenar libros y usuarios.

5. Lectura y Escritura de Archivos con Objetos

La lectura y escritura de archivos con objetos permite almacenar información de manera permanente en archivos.

En Java, este proceso se realiza mediante serialización, permitiendo guardar objetos completos y recuperarlos posteriormente.

Definición

La serialización es el proceso de convertir un objeto en una secuencia de bytes para almacenarlo en un archivo.

Posteriormente, el objeto puede recuperarse mediante deserialización.

Características

  • Permite guardar objetos completos.
  • Facilita la persistencia de datos.
  • Reduce la pérdida de información.
  • Mejora la organización de sistemas.
  • Utiliza clases especializadas de Java IO.

Ejemplo Resuelto


import java.io.FileOutputStream;
import java.io.ObjectOutputStream;
import java.io.Serializable;

class Estudiante implements Serializable {

    String nombre;

    public Estudiante(String nombre) {

        this.nombre = nombre;
    }
}

public class Main {

    public static void main(String[] args) {

        try {

            Estudiante estudiante =
                new Estudiante("Juan");

            FileOutputStream archivo =
                new FileOutputStream("datos.dat");

            ObjectOutputStream salida =
                new ObjectOutputStream(archivo);

            salida.writeObject(estudiante);

            salida.close();

            System.out.println("Objeto guardado");

        } catch (Exception e) {

            System.out.println("Error al guardar");
        }
    }
}

Explicación del Ejemplo

La clase Estudiante implementa Serializable, permitiendo convertir el objeto en bytes.

ObjectOutputStream se utiliza para escribir el objeto dentro de un archivo.

Gracias a esto, la información puede almacenarse de manera permanente.

Ventajas

  • Permite guardar información del sistema.
  • Facilita la recuperación de datos.
  • Mejora la persistencia de aplicaciones.
  • Permite trabajar directamente con objetos.
  • Reduce el uso de variables temporales.

Relación con POO

Este tema se relaciona con POO porque se trabaja directamente con objetos creados a partir de clases.

La serialización permite almacenar el estado completo de un objeto.

Esto demuestra reutilización, encapsulamiento y manejo de entidades orientadas a objetos.

Ejercicios Propuestos

  • Crear un sistema que guarde objetos de una clase Producto en un archivo.
  • Diseñar un programa que lea información de estudiantes desde un archivo serializado.

6. Buenas Prácticas, PEP8 y Refactorización

Las buenas prácticas de programación permiten desarrollar software más organizado, legible y mantenible.

La guía PEP8 establece recomendaciones de estilo para escribir código limpio, mientras que la refactorización consiste en mejorar el código sin alterar su funcionamiento.

Definición

Las buenas prácticas son técnicas y recomendaciones utilizadas para mejorar la calidad del software.

La refactorización reorganiza el código para hacerlo más claro, reutilizable y fácil de mantener.

Características

  • Mejoran la legibilidad del código.
  • Facilitan el mantenimiento del sistema.
  • Permiten detectar errores más fácilmente.
  • Favorecen la reutilización de código.
  • Promueven nombres claros y organizados.

Ejemplo Resuelto


// Codigo sin buenas practicas

class calc {

    int s(int a,int b){

        return a+b;
    }
}

// Codigo refactorizado

class Calculadora {

    int sumar(int numero1, int numero2) {

        return numero1 + numero2;
    }
}

Explicación del Ejemplo

En el primer ejemplo el código utiliza nombres poco claros y mala organización.

En el segundo ejemplo se aplican buenas prácticas, utilizando nombres descriptivos y mejor estructura.

Esto facilita la comprensión y mantenimiento del software.

Ventajas

  • Mayor claridad del código.
  • Facilidad para trabajar en equipo.
  • Reducción de errores.
  • Mayor reutilización de componentes.
  • Mejor mantenimiento del sistema.

Relación con POO

Las buenas prácticas mejoran la implementación de principios de Programación Orientada a Objetos.

Permiten diseñar clases más organizadas, reutilizables y fáciles de mantener.

La refactorización ayuda a optimizar sistemas orientados a objetos sin modificar su comportamiento.

Ejercicios Propuestos

  • Refactorizar una clase con nombres de variables poco descriptivos.
  • Aplicar buenas prácticas a un sistema de gestión de estudiantes.

7. Introducción a la POO y Entorno de Desarrollo en Java

La Programación Orientada a Objetos (POO) es un paradigma de programación que organiza el software mediante clases y objetos, permitiendo desarrollar sistemas más reutilizables, organizados y escalables.

Java es uno de los lenguajes más utilizados para aplicar POO, gracias a sus herramientas de desarrollo, bibliotecas y compatibilidad multiplataforma.

Definición

La POO se basa en representar entidades del mundo real mediante objetos que poseen atributos y métodos.

Para desarrollar aplicaciones Java se utilizan entornos de desarrollo (IDE) y componentes fundamentales como JDK y JVM.

Entornos de Desarrollo en Java

  • IntelliJ IDEA: IDE moderno y ampliamente utilizado para Java.
  • Eclipse: entorno flexible y de código abierto.
  • NetBeans: IDE sencillo orientado a proyectos Java.
  • VS Code: editor ligero compatible con Java mediante extensiones.

JDK, JVM y Paquetes

  • JDK (Java Development Kit): conjunto de herramientas necesarias para desarrollar programas Java.
  • JVM (Java Virtual Machine): máquina virtual encargada de ejecutar programas Java.
  • Paquetes: mecanismo utilizado para organizar clases y evitar conflictos de nombres.

Características de la POO

  • Uso de clases y objetos.
  • Aplicación de encapsulamiento.
  • Permite herencia y polimorfismo.
  • Favorece la reutilización del código.
  • Facilita el mantenimiento del software.

Ejemplo Resuelto


package proyecto;

class Persona {

    String nombre;

    void saludar() {

        System.out.println("Hola " + nombre);
    }
}

public class Main {

    public static void main(String[] args) {

        Persona persona = new Persona();

        persona.nombre = "Juan";

        persona.saludar();
    }
}

Explicación del Ejemplo

El ejemplo muestra una clase llamada Persona con un atributo y un método.

Además, se utiliza un paquete para organizar correctamente el proyecto Java.

Esto demuestra cómo la POO permite representar objetos del mundo real mediante clases.

Ventajas

  • Mayor organización del código.
  • Facilidad para reutilizar clases.
  • Mejor mantenimiento del software.
  • Compatibilidad multiplataforma con JVM.
  • Mayor escalabilidad de proyectos.

Relación con POO

Este tema introduce los conceptos fundamentales necesarios para comprender la Programación Orientada a Objetos.

Además, explica el entorno utilizado para desarrollar aplicaciones Java orientadas a objetos.

Ejercicios Propuestos

  • Crear una clase Estudiante dentro de un paquete llamado universidad.
  • Investigar las diferencias entre JDK, JRE y JVM.

8. Clases y Objetos en Java

Las clases y los objetos son la base fundamental de la Programación Orientada a Objetos (POO).

Una clase es una plantilla que define atributos y métodos, mientras que un objeto es una instancia real creada a partir de esa clase.

Definición

En Java, una clase permite representar entidades del mundo real mediante atributos y comportamientos.

Los objetos contienen valores propios y pueden interactuar entre sí, permitiendo construir programas organizados, reutilizables y escalables.

Características Principales

  • Las clases funcionan como plantillas para crear objetos.
  • Los objetos poseen atributos y métodos propios.
  • Permiten aplicar encapsulamiento y reutilización.
  • Facilitan la organización y mantenimiento del código.
  • Son fundamentales en todos los pilares de la POO.

Estructura Básica de una Clase


class Persona {

    String nombre;
    int edad;

    void saludar() {

        System.out.println("Hola, soy " + nombre);

    }

}

Ejemplo Resuelto


class Estudiante {

    String nombre;
    int semestre;

    void mostrarDatos() {

        System.out.println("Nombre: " + nombre);
        System.out.println("Semestre: " + semestre);

    }

}

public class Main {

    public static void main(String[] args) {

        Estudiante estudiante1 = new Estudiante();

        estudiante1.nombre = "Carlos";
        estudiante1.semestre = 3;

        estudiante1.mostrarDatos();

    }

}

Explicación del Ejemplo

En este ejemplo se crea la clase Estudiante, la cual contiene atributos y un método.

Posteriormente se crea un objeto llamado estudiante1, al cual se le asignan valores.

Finalmente, el método mostrarDatos() imprime la información del objeto.

Importancia en la POO

Las clases y objetos permiten modelar sistemas reales mediante entidades independientes y organizadas.

Gracias a esto, Java facilita el desarrollo de aplicaciones mantenibles, reutilizables y escalables.

Ejercicios Propuestos

  • Crear una clase llamada Libro con atributos título y autor.
  • Crear un objeto de la clase Vehiculo y mostrar su información.
  • Diseñar una clase Empleado con un método para mostrar salario.
