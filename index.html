1. Clases Abstractas e Interfaces

Las clases abstractas e interfaces son elementos fundamentales en la Programación Orientada a Objetos (POO), ya que permiten definir estructuras generales para diferentes clases.

Gracias a estos conceptos, Java facilita la reutilización de código, la modularidad y la implementación de comportamientos comunes en distintas clases.

¿Qué es una Clase Abstracta?

Una clase abstracta es una clase que no puede ser instanciada directamente y que sirve como base para otras clases.

Puede contener métodos normales y métodos abstractos, los cuales deben ser implementados por las subclases.

¿Qué es una Interfaz?

Una interfaz es una estructura que define métodos que otras clases deben implementar.

Las interfaces permiten establecer comportamientos comunes entre diferentes clases, favoreciendo el polimorfismo y la flexibilidad del sistema.

Características Principales

  • Las clases abstractas no pueden crear objetos directamente.
  • Las interfaces definen comportamientos obligatorios.
  • Favorecen la reutilización de código.
  • Mejoran la organización y escalabilidad del software.
  • Permiten aplicar polimorfismo en Java.

Ejemplo Resuelto


abstract class Animal {

    abstract void hacerSonido();

}

interface Mascota {

    void jugar();

}

class Perro extends Animal implements Mascota {

    @Override
    void hacerSonido() {

        System.out.println("El perro ladra");

    }

    @Override
    public void jugar() {

        System.out.println("El perro juega");

    }

}

public class Main {

    public static void main(String[] args) {

        Perro perro = new Perro();

        perro.hacerSonido();

        perro.jugar();

    }

}

Explicación del Ejemplo

La clase abstracta Animal define el método abstracto hacerSonido().

La interfaz Mascota obliga a implementar el método jugar().

La clase Perro hereda de la clase abstracta e implementa la interfaz, demostrando reutilización, herencia y polimorfismo.

Importancia en la POO

  • Permiten crear sistemas más flexibles.
  • Favorecen la reutilización de código.
  • Mejoran la organización de proyectos.
  • Facilitan el mantenimiento del software.
  • Ayudan a implementar polimorfismo.

Ejercicios Propuestos

  • Crear una clase abstracta Figura con un método calcularArea().
  • Diseñar una interfaz Vehiculo con el método acelerar().
  • Implementar una clase Carro utilizando herencia e interfaces.

2. Composición y Modularización del Código

La composición es un principio de la Programación Orientada a Objetos donde una clase contiene objetos de otras clases para construir funcionalidades más complejas.

La modularización consiste en dividir un sistema en pequeñas partes independientes llamadas módulos, facilitando la organización, reutilización y mantenimiento del código.

Definición

En POO, la composición representa una relación del tipo "tiene un". Por ejemplo, un automóvil tiene un motor. Esto permite crear sistemas más organizados y reutilizables.

Características

  • Favorece la reutilización del código.
  • Permite construir sistemas complejos.
  • Mejora la organización del software.
  • Reduce la dependencia entre clases.
  • Facilita el mantenimiento del código.

Ejemplo Resuelto


class Motor {

    void encender() {

        System.out.println("Motor encendido");
    }
}

class Auto {

    private Motor motor;

    public Auto() {

        motor = new Motor();
    }

    void arrancar() {

        motor.encender();

        System.out.println("Auto en movimiento");
    }
}

public class Main {

    public static void main(String[] args) {

        Auto auto = new Auto();

        auto.arrancar();
    }
}

Explicación del Ejemplo

La clase Auto utiliza un objeto de la clase Motor. Esto demuestra composición porque el automóvil "tiene un" motor.

Gracias a este principio de POO, el sistema queda más organizado y cada clase tiene una responsabilidad específica.

Ventajas

  • Mayor reutilización de clases.
  • Mejor mantenimiento del sistema.
  • Mayor escalabilidad del software.
  • Separación clara de responsabilidades.

Ejercicios Propuestos

  • Crear una clase Computador que tenga una clase Procesador utilizando la composición.
  • Diseñar un sistema de biblioteca usando clases Libro, Usuario y Prestamo.

3. Manejo de Errores y Excepciones

El manejo de errores y excepciones es fundamental en la Programación Orientada a Objetos porque permite controlar situaciones inesperadas durante la ejecución del programa.

En Java, las excepciones ayudan a evitar que el sistema finalice de forma incorrecta, mejorando la estabilidad y seguridad del software.

Definición

Una excepción es un evento que ocurre durante la ejecución de un programa y altera el flujo normal de las instrucciones.

Java utiliza las palabras reservadas try, catch y finally para manejar excepciones.

Características

  • Permiten controlar errores de ejecución.
  • Evitan que el programa termine inesperadamente.
  • Facilitan la depuración del sistema.
  • Mejoran la seguridad y estabilidad del software.
  • Permiten separar la lógica del manejo de errores.

Ejemplo Resuelto


public class Main {

    public static void main(String[] args) {

        try {

            int resultado = 10 / 0;

            System.out.println(resultado);

        } catch (ArithmeticException e) {

            System.out.println("Error: division por cero");

        } finally {

            System.out.println("Fin del programa");
        }
    }
}

Explicación del Ejemplo

En el ejemplo se intenta realizar una división entre cero, lo que genera una excepción de tipo ArithmeticException.

El bloque catch captura el error y evita que el programa termine de manera abrupta.

El bloque finally se ejecuta siempre, independientemente de si ocurre o no una excepción.

Ventajas

  • Mayor estabilidad del software.
  • Mejor experiencia para el usuario.
  • Facilita el mantenimiento del sistema.
  • Permite detectar errores de manera controlada.
  • Mejora la calidad del código.

Relación con POO

El manejo de excepciones se relaciona con POO porque las excepciones en Java son objetos que pertenecen a clases específicas.

Además, permite crear sistemas más robustos, organizados y mantenibles.

Ejercicios Propuestos

  • Crear un programa que capture un error al ingresar texto en una variable numérica.
  • Diseñar un sistema que valide si un usuario ingresó correctamente una contraseña.

4. Colecciones y Estructuras de Datos

Las colecciones y estructuras de datos permiten almacenar, organizar y manipular grandes cantidades de información dentro de un programa.

En Java, las colecciones forman parte del Java Collections Framework, el cual proporciona clases y métodos reutilizables para trabajar con datos de manera eficiente.

Definición

Una colección es un objeto que permite almacenar múltiples elementos en una sola estructura.

Algunas colecciones comunes en Java son: ArrayList, LinkedList, HashSet y HashMap.

Características

  • Permiten almacenar múltiples objetos.
  • Facilitan la búsqueda y organización de datos.
  • Son dinámicas y flexibles.
  • Reducen la complejidad del código.
  • Forman parte de la biblioteca estándar de Java.

Ejemplo Resuelto


ArrayList<String> estudiantes = new ArrayList<>();

estudiantes.add("Ana");
estudiantes.add("Carlos");

for (String nombre : estudiantes) {

    System.out.println(nombre);

}

Explicación del Ejemplo

En este ejemplo se utiliza un ArrayList para almacenar nombres de estudiantes.

El método add() permite agregar elementos a la colección, mientras que el ciclo for-each recorre cada objeto almacenado.

Esto demuestra cómo las colecciones facilitan el manejo dinámico de datos en Java.

Ventajas

  • Mayor organización de datos.
  • Facilidad para agregar y eliminar elementos.
  • Mayor reutilización de código.
  • Mejor rendimiento en aplicaciones grandes.
  • Facilitan el desarrollo de sistemas complejos.

Relación con POO

Las colecciones trabajan directamente con objetos, lo cual es fundamental en Programación Orientada a Objetos.

Por ejemplo, un ArrayList puede almacenar objetos creados a partir de una clase específica.

Esto permite organizar entidades del sistema de manera eficiente y reutilizable.

Ejercicios Propuestos

  • Crear un ArrayList que almacene objetos de una clase Producto.

5. Lectura y Escritura de Archivos con Objetos

La lectura y escritura de archivos con objetos permite almacenar información de manera permanente en archivos.

En Java, este proceso se realiza mediante serialización, permitiendo guardar objetos completos y recuperarlos posteriormente.

Definición

La serialización es el proceso de convertir un objeto en una secuencia de bytes para almacenarlo en un archivo.

Posteriormente, el objeto puede recuperarse mediante deserialización.

Características

  • Permite guardar objetos completos.
  • Facilita la persistencia de datos.
  • Reduce la pérdida de información.
  • Mejora la organización de sistemas.
  • Utiliza clases especializadas de Java IO.

Ejemplo Resuelto


import java.io.FileOutputStream;
import java.io.ObjectOutputStream;
import java.io.Serializable;

class Estudiante implements Serializable {

    String nombre;

    public Estudiante(String nombre) {

        this.nombre = nombre;
    }
}

public class Main {

    public static void main(String[] args) {

        try {

            Estudiante estudiante =
                new Estudiante("Juan");

            FileOutputStream archivo =
                new FileOutputStream("datos.dat");

            ObjectOutputStream salida =
                new ObjectOutputStream(archivo);

            salida.writeObject(estudiante);

            salida.close();

            System.out.println("Objeto guardado");

        } catch (Exception e) {

            System.out.println("Error al guardar");
        }
    }
}

Explicación del Ejemplo

La clase Estudiante implementa Serializable, permitiendo convertir el objeto en bytes.

ObjectOutputStream se utiliza para escribir el objeto dentro de un archivo.

Gracias a esto, la información puede almacenarse de manera permanente.

Ventajas

  • Permite guardar información del sistema.
  • Facilita la recuperación de datos.
  • Mejora la persistencia de aplicaciones.
  • Permite trabajar directamente con objetos.
  • Reduce el uso de variables temporales.

Relación con POO

Este tema se relaciona con POO porque se trabaja directamente con objetos creados a partir de clases.

La serialización permite almacenar el estado completo de un objeto.

Esto demuestra reutilización, encapsulamiento y manejo de entidades orientadas a objetos.

Ejercicios Propuestos

  • Crear un sistema que guarde objetos de una clase Producto en un archivo.
  • Diseñar un programa que lea información de estudiantes desde un archivo serializado.

7. Introducción a la POO y Entorno de Desarrollo en Java

La Programación Orientada a Objetos (POO) es un paradigma de programación que organiza el software mediante clases y objetos, permitiendo desarrollar sistemas más reutilizables, organizados y escalables.

Java es uno de los lenguajes más utilizados para aplicar POO, gracias a sus herramientas de desarrollo, bibliotecas y compatibilidad multiplataforma.

Definición

La POO se basa en representar entidades del mundo real mediante objetos que poseen atributos y métodos.

Para desarrollar aplicaciones Java se utilizan entornos de desarrollo (IDE) y componentes fundamentales como JDK y JVM.

Entornos de Desarrollo en Java

  • IntelliJ IDEA: IDE moderno y ampliamente utilizado para Java.
  • Eclipse: entorno flexible y de código abierto.
  • NetBeans: IDE sencillo orientado a proyectos Java.
  • VS Code: editor ligero compatible con Java mediante extensiones.

JDK, JVM y Paquetes

  • JDK (Java Development Kit): conjunto de herramientas necesarias para desarrollar programas Java.
  • JVM (Java Virtual Machine): máquina virtual encargada de ejecutar programas Java.
  • Paquetes: mecanismo utilizado para organizar clases y evitar conflictos de nombres.

Características de la POO

  • Uso de clases y objetos.
  • Aplicación de encapsulamiento.
  • Permite herencia y polimorfismo.
  • Favorece la reutilización del código.
  • Facilita el mantenimiento del software.

Ejemplo Resuelto


package proyecto;

class Persona {

}

public class Main {

        Persona persona = new Persona();

        persona.nombre = "Juan";

        persona.saludar();
    }
}

Explicación del Ejemplo

El ejemplo muestra una clase llamada Persona con un atributo y un método.

Además, se utiliza un paquete para organizar correctamente el proyecto Java.

Esto demuestra cómo la POO permite representar objetos del mundo real mediante clases.

Ventajas

  • Mayor organización del código.
  • Facilidad para reutilizar clases.
  • Mejor mantenimiento del software.
  • Compatibilidad multiplataforma con JVM.
  • Mayor escalabilidad de proyectos.

Relación con POO

Este tema introduce los conceptos fundamentales necesarios para comprender la Programación Orientada a Objetos.

8. Clases y Objetos en Java

Las clases y los objetos son la base fundamental de la Programación Orientada a Objetos (POO).

Una clase es una plantilla que define atributos y métodos, mientras que un objeto es una instancia real creada a partir de esa clase.

Definición

En Java, una clase permite representar entidades del mundo real mediante atributos y comportamientos.

Los objetos contienen valores propios y pueden interactuar entre sí, permitiendo construir programas organizados, reutilizables y escalables.

Características Principales

  • Las clases funcionan como plantillas para crear objetos.
  • Los objetos poseen atributos y métodos propios.
  • Permiten aplicar encapsulamiento y reutilización.
  • Facilitan la organización y mantenimiento del código.
  • Son fundamentales en todos los pilares de la POO.

Estructura Básica de una Clase


class Persona {

    String nombre;
    int edad;

    void saludar() {

        System.out.println("Hola, soy " + nombre);

    }

}

Ejemplo Resuelto


class Estudiante {

    String nombre;
    int semestre;

    void mostrarDatos() {

        System.out.println("Nombre: " + nombre);
        System.out.println("Semestre: " + semestre);

    }

}

public class Main {

    public static void main(String[] args) {

        Estudiante estudiante1 = new Estudiante();

        estudiante1.nombre = "Carlos";
        estudiante1.semestre = 3;

        estudiante1.mostrarDatos();

    }

}

Explicación del Ejemplo

En este ejemplo se crea la clase Estudiante, la cual contiene atributos y un método.

Posteriormente se crea un objeto llamado estudiante1, al cual se le asignan valores.

Finalmente, el método mostrarDatos() imprime la información del objeto.

Importancia en la POO

Las clases y objetos permiten modelar sistemas reales mediante entidades independientes y organizadas.

Gracias a esto, Java facilita el desarrollo de aplicaciones mantenibles, reutilizables y escalables.

Ejercicios Propuestos

  • Crear un objeto de la clase Vehiculo y mostrar su información.
  • Diseñar una clase Empleado con un método para mostrar salario.

9. Atributos, Métodos y Encapsulamiento

En Programación Orientada a Objetos, los atributos representan las características o datos de un objeto, mientras que los métodos representan las acciones o comportamientos que puede realizar.

El encapsulamiento es uno de los pilares fundamentales de la POO. Consiste en proteger los datos de una clase evitando el acceso directo desde otras clases. Para ello se utilizan modificadores de acceso y métodos especiales llamados getters y setters.

¿Qué son los atributos?

Los atributos son variables que almacenan información de un objeto. Por ejemplo, un objeto Persona puede tener:

  • nombre
  • edad
  • correo

¿Qué son los métodos?

Los métodos son funciones definidas dentro de una clase. Permiten ejecutar acciones relacionadas con los objetos.

Ejemplos:

  • mostrarInformacion()
  • calcularEdad()
  • enviarMensaje()

Modificadores de Acceso

Los modificadores de acceso controlan desde dónde se puede acceder a los atributos y métodos de una clase.

  • public: accesible desde cualquier clase.
  • private: solo accesible dentro de la misma clase.
  • protected: accesible en la clase y sus subclases.

Getters y Setters

Los getters permiten obtener el valor de un atributo privado, mientras que los setters permiten modificarlo de manera segura.

Ejemplo Resuelto


class Persona {

    // Atributos privados
    private String nombre;
    private int edad;

    // Setter
    public void setNombre(String nombre) {
        this.nombre = nombre;
    }

    // Getter
    public String getNombre() {
        return nombre;
    }

    // Setter
    public void setEdad(int edad) {
        this.edad = edad;
    }

    // Getter
    public int getEdad() {
        return edad;
    }

    // Método
    public void mostrarDatos() {
        System.out.println("Nombre: " + nombre);
        System.out.println("Edad: " + edad);
    }
}

Explicación del Ejemplo

En este ejemplo los atributos nombre y edad son privados, por lo que no pueden modificarse directamente desde otra clase.

Para acceder a ellos se utilizan los métodos getNombre() y getEdad().

Para modificar sus valores se usan los métodos setNombre() y setEdad().

Esto permite proteger la información y controlar cómo se manipulan los datos dentro del programa.

Importancia del Encapsulamiento

  • Protege los datos de modificaciones incorrectas.
  • Mejora la seguridad del programa.
  • Facilita el mantenimiento del código.
  • Permite controlar el acceso a la información.
  • Hace que las clases sean más organizadas y reutilizables.

10. Constructores y Representación de Objetos

Los constructores son métodos especiales utilizados para inicializar objetos en Java. Permiten asignar valores iniciales a los atributos cuando se crea un objeto.

Además, Java utiliza el método toString() para representar objetos en forma de texto y la sobrecarga permite crear múltiples métodos o constructores con diferentes parámetros.

¿Qué es un Constructor?

Un constructor es un método especial que se ejecuta automáticamente al crear un objeto.

Su nombre debe ser igual al de la clase y no posee tipo de retorno.

Características de los Constructores

  • Se ejecutan automáticamente al crear objetos.
  • No tienen tipo de retorno.
  • Poseen el mismo nombre de la clase.
  • Pueden recibir parámetros.
  • Permiten inicializar atributos.

Método toString()

El método toString() permite representar un objeto como una cadena de texto.

Es útil para mostrar información de manera organizada y legible.

Sobrecarga de Métodos y Constructores

La sobrecarga consiste en crear múltiples métodos o constructores con el mismo nombre pero diferentes parámetros.

Esto permite reutilizar código y mejorar la flexibilidad de las clases.

Ejemplo Resuelto


class Estudiante {

    String nombre;
    int edad;

    // Constructor vacio
    public Estudiante() {

        nombre = "Sin nombre";
        edad = 0;
    }

    // Constructor con parametros
    public Estudiante(String nombre, int edad) {

        this.nombre = nombre;
        this.edad = edad;
    }

    // Metodo toString
    public String toString() {

        return "Nombre: " + nombre +
               " Edad: " + edad;
    }

}

public class Main {

    public static void main(String[] args) {

        Estudiante estudiante1 =
            new Estudiante();

        Estudiante estudiante2 =
            new Estudiante("Carlos", 20);

        System.out.println(estudiante1);

        System.out.println(estudiante2);

    }

}

Explicación del Ejemplo

La clase Estudiante posee dos constructores: uno vacío y otro con parámetros.

Esto demuestra la sobrecarga de constructores, permitiendo crear objetos de diferentes maneras.

Además, el método toString() devuelve una representación textual del objeto.

Importancia en la POO

  • Facilitan la creación de objetos.
  • Permiten inicializar atributos correctamente.
  • Mejoran la reutilización de código.
  • Hacen más legible la representación de objetos.
  • Favorecen la flexibilidad mediante sobrecarga.

Ejercicios Propuestos

  • Crear una clase Libro con dos constructores diferentes.
  • Implementar el método toString() en una clase Vehiculo.
  • Diseñar una clase Producto utilizando sobrecarga.

11. Herencia en Java

La herencia es uno de los pilares fundamentales de la Programación Orientada a Objetos (POO).

Permite que una clase herede atributos y métodos de otra clase, favoreciendo la reutilización de código y la organización de sistemas complejos.

¿Qué es la Herencia?

La herencia consiste en crear una nueva clase basada en otra clase existente.

La clase que hereda se denomina subclase o clase hija, mientras que la clase original se llama superclase o clase padre.

Herencia Simple

Java utiliza herencia simple, lo que significa que una clase solo puede heredar directamente de una sola clase.

Para heredar se utiliza la palabra reservada extends.

Uso de super()

La palabra reservada super permite acceder a atributos, métodos y constructores de la clase padre.

Es muy utilizada para reutilizar inicializaciones y evitar duplicación de código.

Características de la Herencia

  • Permite reutilizar código.
  • Facilita la organización de clases.
  • Reduce la duplicación de atributos y métodos.
  • Favorece la escalabilidad de programas.
  • Es fundamental para el polimorfismo.

Ejemplo Resuelto


class Persona {

    String nombre;

    public Persona(String nombre) {

        this.nombre = nombre;
    }

    void mostrarNombre() {

        System.out.println("Nombre: " + nombre);
    }

}

class Estudiante extends Persona {

    int semestre;

    public Estudiante(String nombre, int semestre) {

        super(nombre);

        this.semestre = semestre;
    }

    void mostrarSemestre() {

        System.out.println("Semestre: " + semestre);
    }

}

public class Main {

    public static void main(String[] args) {

        Estudiante estudiante =
            new Estudiante("Carlos", 3);

        estudiante.mostrarNombre();

        estudiante.mostrarSemestre();

    }

}

Explicación del Ejemplo

La clase Estudiante hereda de la clase Persona.

12. Polimorfismo y Sobreescritura de Métodos

El polimorfismo es uno de los pilares fundamentales de la Programación Orientada a Objetos (POO).

Permite que un mismo método tenga diferentes comportamientos dependiendo del objeto que lo utilice.

En Java, el polimorfismo se aplica principalmente mediante la herencia y la sobreescritura de métodos.

¿Qué es el Polimorfismo?

El término polimorfismo significa "muchas formas".

En POO, permite que diferentes clases respondan de manera distinta al mismo método.

Sobreescritura de Métodos

La sobreescritura ocurre cuando una subclase redefine un método heredado de la clase padre.

Esto permite adaptar comportamientos específicos en cada objeto.

Polimorfismo Dinámico

El polimorfismo dinámico ocurre cuando el método que se ejecuta se determina en tiempo de ejecución.

Esto sucede cuando una referencia de la clase padre apunta a objetos de diferentes subclases.

Características

  • Permite reutilización de código.
  • Facilita la flexibilidad de programas.
  • Reduce la complejidad del sistema.
  • Trabaja junto con la herencia.
  • Mejora la escalabilidad del software.

Ejemplo Resuelto


class Animal {

    void hacerSonido() {

        System.out.println("El animal hace un sonido");
    }

}

class Perro extends Animal {

    @Override
    void hacerSonido() {

        System.out.println("El perro ladra");
    }

}

class Gato extends Animal {

    @Override
    void hacerSonido() {

        System.out.println("El gato maulla");
    }

}

public class Main {

    public static void main(String[] args) {

        Animal animal1 = new Perro();

        Animal animal2 = new Gato();

        animal1.hacerSonido();

        animal2.hacerSonido();

    }

}

Explicación del Ejemplo

La clase Animal define el método hacerSonido().

Las clases Perro y Gato sobrescriben ese método con comportamientos diferentes.

Aunque las variables son de tipo Animal, Java ejecuta el método correspondiente al objeto real.

Esto demuestra el polimorfismo dinámico.

Importancia en la POO

  • Permite programas más flexibles.
  • Facilita la reutilización de métodos.
  • Reduce duplicación de código.
  • Mejora la organización del sistema.
  • Favorece la extensibilidad de aplicaciones.

Ejercicios Propuestos

  • Crear una clase Figura y subclases Circulo y Rectangulo.
  • Implementar polimorfismo con una clase Vehiculo y subclases Carro y Moto.
  • Sobrescribir un método mostrarInformacion() en diferentes clases.
