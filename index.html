Manual web académico sobre los temas fundamentales de POO en Java.

1. Clases y Objetos

Las clases y los objetos son pilares fundamentales de la Programación Orientada a Objetos. Una clase funciona como una plantilla que define atributos y comportamientos, mientras que un objeto es una instancia creada a partir de esa clase.

En Java, las clases permiten representar entidades del mundo real mediante propiedades y métodos, facilitando la organización y reutilización del código.

Ejemplo 1: Clase Persona


class Persona {

    String nombre;
    int edad;

    void saludar() {
        System.out.println("Hola, soy " + nombre);
    }
}

public class Main {

    public static void main(String[] args) {

        Persona persona1 = new Persona();

        persona1.nombre = "Juan";
        persona1.edad = 20;

        persona1.saludar();
    }
}

Ejemplo 2: Clase Vehiculo


class Vehiculo {

    String marca;
    int velocidad;

    void acelerar() {
        velocidad += 10;
    }
}

public class Main {

    public static void main(String[] args) {

        Vehiculo carro = new Vehiculo();

        carro.marca = "Toyota";
        carro.velocidad = 50;

        carro.acelerar();

        System.out.println(carro.velocidad);
    }
}

2. Composición y Modularización del Código

La composición es una técnica de Programación Orientada a Objetos donde una clase contiene objetos de otras clases para construir funcionalidades más complejas.

La modularización permite dividir el código en partes pequeñas, organizadas y reutilizables, facilitando el mantenimiento y escalabilidad del software.

Ejemplo 1: Composición entre Auto y Motor


class Motor {

    void encender() {
        System.out.println("Motor encendido");
    }
}

class Auto {

    Motor motor = new Motor();

    void arrancar() {
        motor.encender();
        System.out.println("Auto en movimiento");
    }
}

public class Main {

    public static void main(String[] args) {

        Auto auto = new Auto();

        auto.arrancar();
    }
}

Ejemplo 2: Modularización en una Biblioteca


class Libro {

    String titulo;
}

class Usuario {

    String nombre;
}

class Prestamo {

    Libro libro;
    Usuario usuario;
}

public class Main {

    public static void main(String[] args) {

        Libro libro = new Libro();
        Usuario usuario = new Usuario();

        Prestamo prestamo = new Prestamo();

        prestamo.libro = libro;
        prestamo.usuario = usuario;

        System.out.println("Préstamo realizado");
    }
}

Ejercicios Propuestos

  • Crear una clase Computador que tenga una clase Procesador utilizando composición.
  • Diseñar un sistema modular para una biblioteca utilizando clases separadas para Libro, Usuario y Prestamo.
